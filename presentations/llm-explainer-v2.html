What is a Neural Network?

The foundational architecture behind every modern language model

A neural network is a mathematical system composed of layers of simple computational units. Rather than encoding hand-written rules, it learns patterns from data — adjusting millions of internal parameters until its outputs match the training examples it has seen.

For text, this means a network can learn that the words "failed," "despite," and "outcomes" co-occur in contexts expressing accountability failure — without ever being told what those words mean.

Core intuition Teaching a child to recognise cats works not by giving them a rulebook, but by repeated exposure. Neural networks learn the same way — from millions of examples — except the "understanding" is encoded as numerical weights rather than mental concepts.

Layered architecture

Every network has three functional zones: an input layer (raw data as numbers), one or more hidden layers (where patterns are extracted progressively), and an output layer (a prediction or classification).

Why does this matter for text? Text must first be converted into numbers — through embeddings, a kind of high-dimensional map where semantically similar words sit close together. The network then learns the statistical structure of language across this numeric landscape.

The Individual Neuron

Every billion-parameter model is built from iterations of this single unit

A single artificial neuron receives several numeric inputs, multiplies each by a weight (its learned importance), sums everything up, and produces an output via an activation function. That activation decides whether — and how strongly — the signal propagates forward.

Analogy for researchers A peer-review committee scoring a grant application. Each criterion (impact, feasibility, originality) receives a different weight from each reviewer. The weighted total determines whether funding is recommended. Training the network means learning which weights produce the best decisions.

Learning = adjusting the weights

When the network predicts wrong, backpropagation calculates each weight's contribution to the error and nudges every weight slightly in the right direction — a process called gradient descent. Repeated across millions of examples, the network converges on a useful solution.

What a neuron does

  • Receives multiple numeric inputs
  • Multiplies each by a learned weight
  • Sums them with a bias term
  • Passes through an activation function
  • Sends output to the next layer

What training does

  • Feeds data through the network
  • Compares output to ground truth
  • Computes the loss (error)
  • Backpropagates gradients
  • Repeats for millions of examples

Transformers

The 2017 architecture that made modern LLMs possible — step by step

Before 2017, neural networks processed text sequentially — one word at a time — struggling with long documents and distant dependencies. Google's "Attention Is All You Need" replaced recurrence with a mechanism that processes all tokens simultaneously, attending to any position in the sequence regardless of distance.

Core innovation Reading a policy document, a researcher simultaneously connects "beneficiaries" to funding figures, outcomes, and geography — wherever they appear. A transformer does this computationally, for every token against every other token, in parallel.

BERT vs GPT BERT uses a bidirectional encoder — every token attends to all others simultaneously. Ideal for classification, entity extraction, and understanding tasks. GPT is a unidirectional decoder — predicts the next token from prior context only. Best for text generation. Most social science text analysis tools are BERT-derived.

Self-Attention in Action

The mechanism that allows a model to contextualise every token against every other

For each token, self-attention asks: which other tokens in this sequence are most relevant for understanding this one? The answer is a set of attention scores — used to build a contextualised representation that is unique to its surrounding context.

The word "bank" attended to "river" gets a very different representation than "bank" attended to "loan" — this is how transformers disambiguate meaning that bag-of-words models cannot.

Research implication In a corpus of charity impact reports, attention allows the model to connect "vulnerable" to "communities," "services," and "funding" — extracting relational meaning that keyword frequency analysis cannot capture. This underpins why transformer-based classifiers substantially outperform TF-IDF baselines on specialised text.

Multi-head attention

Transformers run multiple attention operations in parallel (each called a "head"), each capable of capturing a different relationship type: syntactic agreement, semantic proximity, coreference, contrast, and so on. GPT-4 uses 128 heads per layer across 96 layers — providing enormous representational capacity.

What attention captures

  • Pronoun–referent resolution
  • Negation and modifier scope
  • Sentiment polarity over distance
  • Domain-specific co-occurrence
  • Contrast and causal links

What attention does not do

  • It does not "read" like a human
  • It has no persistent memory
  • It learns correlation, not causation
  • It can fail on out-of-distribution text
  • It cannot verify factual claims

LLMs in Social Science Research

Practical use cases — click each card for detail, tools, and a worked example

LLMs function as scalable text analysis instruments — automating tasks that previously required thousands of hours of manual coding. For civil society, policy, and public sector research, the relevant applications span the full analytical pipeline.

Key methodological challenges

⚠️ Validity

LLMs trained on general web text may perform poorly on specialist language — third-sector jargon, regulatory filings, policy discourse. Always validate against a human-coded gold standard before deployment.

🔍 Interpretability

Unlike regression coefficients, attention weights and embeddings are not directly interpretable. Tools like SHAP, LIME, and BERTViz provide partial explanations but do not substitute for construct validity.

⚖️ Fine-tune vs prompt

Fine-tuning on labelled data gives more reliable results on narrow tasks. Zero/few-shot prompting is faster and cheaper but less consistent. The tradeoff depends on dataset size and task specificity.

Worked example — CIC report analysis A researcher with a corpus of 50,000 CIC annual reports could use a fine-tuned BERT classifier to categorise each report's impact claims against IMP dimensions; extract financial uncertainty language via a second classification head; and embed each report as a vector to cluster organisations by thematic proximity — all without manual coding.
